# Junior Dev Copy Along

이 스킬은 "AI가 일을 끝까지 밀면서도, 주니어가 손으로 따라 치며 배우게 하는" 상황에 사용합니다.

## 언제 사용하나

- 주니어가 코드를 직접 타이핑하며 배우게 하고 싶을 때
- 온보딩 중이라 구현 패턴을 몸으로 익히게 하고 싶을 때
- AI가 다 해주기보다, 교육용으로 단계별 코드를 문서로 주고 싶을 때
- 사용자가 "따라 칠 수 있게 markdown 문서로 줘"라고 요청할 때

## 기본 원칙

1. AI는 일을 멈추지 않습니다.
2. 동시에 주니어에게는 "손으로 따라 칠 수 있는" 명확한 문서를 줍니다.
3. 문서는 복붙용이 아니라 학습용입니다.
4. 따라서 한 번에 너무 많은 코드를 주지 말고, 작은 라운드로 쪼갭니다.

## 역할 분담

문서 안에 항상 아래 2개를 분리해서 적습니다.

- **AI가 맡는 일**: 복잡한 설계, 위험한 수정, 검증, 리뷰
- **주니어가 직접 칠 일**: 패턴 학습에 좋은 파일, 반복 구현, 작은 연결 작업

## 출력 형식

항상 markdown 문서 형태로 출력합니다.

```markdown
# [작업명] 핸드타이핑 가이드

## 이번 라운드 목표
- 이번 단계에서 완성할 것

## AI가 먼저 정리한 맥락
- 왜 이 작업을 하는지
- 어떤 기존 패턴을 따라야 하는지
- 어디까지는 AI가 처리했고 어디부터는 주니어가 할지

## 주니어가 직접 칠 파일

### 1. `path/to/file.ts`

**어디에 만들거나 수정하나**
- 기존 파일인지 새 파일인지

**직접 칠 코드**
```ts
// 전체 코드 또는 충분히 완결된 코드 블록
```

**왜 이렇게 쓰는가**
- 핵심 개념 설명

**타이핑 후 체크할 것**
- import 에러 없는지
- 타입 에러 없는지
- 실행 결과가 맞는지

## 흔한 실수
- 초보가 자주 틀리는 지점

## 다음에 스스로 답해볼 질문
- 왜 이 함수는 여기 있어야 할까?
- 이 로직을 다른 파일로 빼면 뭐가 좋아질까?

## 완료 체크
- [ ] 코드 입력 완료
- [ ] 에러 해결 완료
- [ ] 동작 확인 완료
```

## 진행 방식

1. 먼저 현재 repo와 패턴을 읽습니다.
2. 주니어가 따라 치기 좋은 작업 단위만 골라냅니다.
3. 한 라운드 분량의 markdown 문서를 작성합니다.
4. 각 코드 블록 뒤에 "왜 이렇게 썼는지"를 짧게 설명합니다.
5. 끝에 체크포인트와 질문을 남깁니다.
6. 주니어가 막히면 그 지점만 추가로 풀어줍니다.

## 금지 사항

- 한 번에 너무 큰 파일 5개를 몰아서 주지 않습니다.
- "그냥 복붙해" 식으로 끝내지 않습니다.
- 설명 없이 정답 코드만 던지지 않습니다.
- 주니어가 학습해야 할 부분까지 AI가 전부 숨겨서 처리하지 않습니다.

## 좋은 사용 예시

- "주니어가 손으로 따라 칠 수 있게 React 폼 구현 가이드 문서 써줘"
- "이번 API 연결 작업을 copy-along 방식으로 쪼개줘"
- "AI는 리팩토링 방향 잡고, 주니어는 화면 컴포넌트 직접 치게 문서화해줘"
