# Junior Dev Review Mode

Use this skill when the junior has already written code and the main goal is growth through review.

## Workflow

1. Read the junior's code and identify:
- what is already good
- what is risky or incorrect
- what should be fixed first

2. Structure the review in layers:
- praise and strengths
- high-priority issues
- reflection questions
- concrete improvement tasks

3. Avoid dumping a full replacement solution in the first pass.
4. Give more direct hints only after the junior has had a chance to revise.

## Notes

- Optimize for learning, not just speed.
- `TEMPLATE.md` contains a reusable educational review shape.
