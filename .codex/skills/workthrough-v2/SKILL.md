# Workthrough V2

Use this skill when the user wants the same post-task documentation idea as `workthrough`, but shorter and in Korean.

## Workflow

1. Summarize the task in a few sentences.
2. List the main changes:
- implemented
- fixed
- improved

3. Include only the most important verification results.
4. Suggest practical next steps.
5. Save under `workthrough/` using a timestamped filename.
