# Junior Dev Spec Coach

Use this skill when the goal is independent problem solving rather than guided transcription.

## Workflow

1. Inspect the repo and define the task clearly.
2. Write a spec that includes:
- goal
- required functionality
- constraints and existing patterns
- suggested file boundaries
- edge cases
- acceptance criteria

3. Give implementation hints without giving final code.
4. Include review questions the junior should answer after coding.
5. Once the junior implements it, switch to review and feedback mode.

## Notes

- Do not give away the final implementation too early.
- Do not stay so abstract that the junior cannot start.
- The sweet spot is a clear spec with enough structure to code independently.
- `TEMPLATE.md` contains a reusable spec shape.
